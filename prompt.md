# Task Prompt

Develop a LocalTrade Marketplace Operations system for an offline-first, on-prem marketplace where sellers publish product listings with media, buyers complete transactions, and both parties build trust through verified reviews. The Angular web app provides English UI for five roles: Buyer, Seller, Moderator (content), Arbitrator (review disputes), and Administrator. Sellers create listings, upload images/videos/attachments with drag-and-drop and resumable uploads (5 MB chunks, up to 20 files per listing, 2 GB per file; JPG/PNG/MP4/PDF only), and see immediate client-side validation for file type and size plus progress and retry controls. After upload, the UI shows extracted metadata (image dimensions, duration, codec) and a “Ready to Publish” gate that can be blocked if the listing is flagged. Buyers place orders, view an offline “payment status” capture (cash, check, store credit, or imported card-terminal settlement), and once an order is marked completed they can leave a review within 14 days with a 1–5 star rating, up to 1,000 characters, and up to 5 images. The storefront displays seller credit metrics including a rolling 90-day average rating and positive-rate percentage, with configurable ranking rules such as “verified purchase first,” “most recent,” or “highest rated,” and clear badges when a review is under appeal or removed by arbitration.

The backend uses Fastify to expose decoupled REST APIs consumed by the Angular frontend, with PostgreSQL as the system of record for users, roles/permissions (RBAC), listings, assets, orders, payments, refunds, reviews, appeals, and audit logs. Media assets are stored in on-prem object storage (local filesystem or NAS path) with signed, time-limited download URLs (default 15 minutes) to prevent hotlinking; transcoding/compression and metadata extraction run locally via asynchronous job queues persisted in PostgreSQL to handle high concurrency without external services. Content safety checks are offline and deterministic: MIME sniffing, extension allow-lists, file fingerprinting to prevent re-uploads of blocked content, and keyword/regex rules for prohibited terms; flagged items are publish-blocked until a Moderator decision with timestamped notes. Payment and refund handling is fully offline: the “gateway” is a pluggable adapter layer that records tender details, enforces idempotency on callbacks/imports using unique transaction keys, and creates reconciliation records from locally imported settlement files; refunds require Seller initiation and Administrator approval over $250.00, then generate a reversal record and a pending/confirmed state based on the same import flow. Security and privacy include least-privilege RBAC, AES-256 encryption for sensitive fields (for example tax IDs and bank routing/account numbers) with masked display (last 4 only), JWT-based API authentication, HMAC-signed webhooks limited to the local network, rate limiting (60 requests/minute per user), anti-injection validation, anti-privilege-escalation checks on every write, anti-replay using nonces and 5-minute timestamp windows, and backups to local encrypted media nightly with 30-day retention and a documented restore target of 4 hours.
